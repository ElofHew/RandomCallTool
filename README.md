# 随机抽取工具

![RandomCallTool](./icon/rctool.ico)

</div>

## 简介

随机抽取工具（RandomCallTool）是一个基于 Python3 + tkinter 的桌面应用程序，主要用于在指定人员名单或组中进行随机抽取。

主程序提供**随机抽组**和**随机抽人**两大功能模块，并内置三档可切换的抽样模式（基本抽样/智能抽样/加权抽样）。配套的**名单编码工具**用于将明文名单编码为 RCP 文件，供主程序的"随机抽人"功能使用。

## 主程序 v2.3

### 主要功能

1. **随机抽组**：支持从数字（1-26）或字母（A-Z）中随机抽取指定数量的组。
2. **随机抽人**：支持从样本文件（`.rcp` / `.txt` / `.csv`）或样本库中随机抽取指定数量的人名。
3. **三档抽样模式**：
   - **基本抽样**（模式 0）：`random.sample` 简单随机抽取。
   - **智能抽样**（模式 1）：追踪近期抽取历史，自动降低刚被选中项的权重，避免连续抽到同一个人。
   - **加权抽样**（模式 2）：可为每个样本单独设置权重，权重越高被抽中的概率越大。
4. **样本库管理**：可将常用名单导入样本库（上限 50 个），支持从样本库快速加载。
5. **整合式操作界面**：抽人与抽组在一个选项卡内通过 RadioButton 切换，交互流畅。
6. **历史记录面板**：右侧面板展示抽取历史，支持单条保存和批量保存全部历史。
7. **配置管理**：多选项卡配置窗口（基本设置、抽样设置、样本管理、默认值、更新设置），支持自动保存结果、保存路径、自动加载样本等选项。
8. **自动检测更新**：启动时静默检测 GitHub/Gitee 新版本，发现新版本时弹窗提示下载。
9. **详细日志记录**：按日期滚动记录程序运行状态，支持查看和清理日志文件。
10. **快捷键操作**：丰富的键盘快捷键支持，提升操作效率。

### 快捷键支持

| 快捷键 | 功能 |
|:---|:---|
| `Ctrl+O` | 选择样本文件 |
| `Ctrl+Shift+O` | 从样本库加载 |
| `Ctrl+R` | 重新加载当前文件 |
| `Ctrl+D` | 自动加载默认样本 |
| `Ctrl+I` | 导入样本到样本库 |
| `Ctrl+Enter` | 执行抽取 |
| `Ctrl+S` | 保存最新结果 |
| `Ctrl+Shift+S` | 批量保存所有历史 |
| `Ctrl+W` | 清除所有历史记录 |
| `Ctrl+Shift+R` | 重置抽样历史 |
| `Ctrl+,` | 打开配置窗口 |
| `Ctrl+T` | 切换到随机抽取选项卡 |
| `Ctrl+L` | 查看日志文件 |

### 使用说明

1. **随机抽组**：
   - 切换到"随机抽取"选项卡，选择"随机抽组"模式。
   - 选择分组方式（数字 123 或字母 ABC）。
   - 设置样本总数（1-26）和要抽取的组数。
   - 点击"抽取"按钮进行随机抽取。
   - 结果展示在程序窗口，并可保存为 HTML 文件。

2. **随机抽人**：
   - 切换到"随机抽取"选项卡，选择"随机抽人"模式。
   - 加载样本文件：支持手动选择文件、从样本库加载、自动加载默认文件或重新加载当前文件。
   - 设置抽取数量，选择抽样模式。
   - 点击"抽取"按钮进行随机抽取。
   - 结果展示在程序窗口，并可保存为 HTML 文件。

3. **配置设置**：
   - 通过菜单栏"编辑 → 配置"打开多选项卡配置窗口。
   - **基本设置**：自动保存结果、结果保存位置、自动加载样本、合并重复名字等。
   - **抽样设置**：默认抽样模式、智能抽样记忆窗口大小。
   - **样本管理**：查看/删除已导入样本库的名单。
   - **默认值**：默认抽取模式、默认抽取数量、默认组总数等。
   - **更新设置**：切换更新源（GitHub/Gitee）、开启/关闭自动检测更新。
   - 点击"确定"或"应用"保存配置。

4. **查看日志**：
   - 通过菜单栏"日志 → 查看日志"打开当日日志文件。
   - 通过"日志 → 清除日志"清理历史日志文件。

5. **检测更新**：
   - 通过菜单栏"工具 → 检测更新"手动检查版本更新。
   - 也可在启动时自动静默检测（可在配置中关闭）。

6. **生成 RCP 文件**：
   - 通过菜单栏"工具 → 生成RCP文件"直接打开配套的名单编码工具。

### 注意事项

1. 支持重复名字自动去重（可在配置中关闭）。
2. 抽取人数不能超过样本总数（自动限制）。
3. 默认样本文件存放在 `data/default.rcp`。
4. 结果文件默认保存在 `data/result` 目录或桌面，具体位置可在配置中设置。
5. 样本库文件存储在 `data/rcplist/` 目录。
6. 支持跨平台运行（Windows / Linux / macOS），但仅提供 Windows 编译版本。

## 随机抽人名单编码工具 v1.1

### 主要功能

1. **编码转换**：支持 **Base64** 和 **Hex** 两种编码方式，将明文名单转换为 RCP 文件。
2. **文本处理**：去除重复项、首尾空格、空行，支持按字母排序。
3. **批量编码/解码**：支持对整个目录下的 `.txt` 文件进行批量编码，支持批量解码测试。
4. **命令行模式**：支持命令行参数直接指定文件、批量目录、编码方式等。
5. **编码测试与验证**：内置解码测试和编码/解码循环验证功能。
6. **快捷键支持**：丰富的键盘快捷键操作（Ctrl+O 打开、Ctrl+S 编码保存、Ctrl+Enter 快速编码等）。
7. **最近文件管理**：记录最近打开的文件列表，快速重新加载。
8. **统计信息**：显示输入文本的行数、字数等统计信息。
9. **配置管理**：通过多选项卡配置窗口设置输出路径、编码方式、文本处理选项等。
10. **窗口状态恢复**：自动保存和恢复窗口位置与大小。
11. **日志记录**：记录程序运行状态，便于调试和查看。

### 使用说明

1. **输入明文名单**：
   - 在上方文本框中输入名单（每行一个名字）。
   - 点击"从文件加载"（Ctrl+O）导入文本文件。
   - 点击"追加文件"可将另一个文件的内容追加到输入区。

2. **处理文本（可选）**：
   - 点击"处理文本"（F5）自动处理：去除重复项、首尾空格、空行等（按配置执行）。

3. **编码名单**：
   - 在底部选择编码方式（Base64 / Hex）。
   - 点击"快速编码"（Ctrl+Enter）仅生成编码后的文本，不保存文件。
   - 点击"编码并保存"（Ctrl+S）编码并保存为 `.rcp` 文件。

4. **输出文件**：
   - 编码结果会显示在下方文本框中。
   - 点击"复制到剪贴板"将编码后的文本复制到剪贴板。
   - 点击"保存为明文"（Ctrl+Shift+S）将输入区文本保存为明文文件。

5. **解码测试**：
   - 点击"解码测试"（Ctrl+D）对输出区内容进行解码验证。
   - 点击"测试编码/解码循环"验证编码功能的完整性。

6. **批量处理**：
   - 通过"文件 → 批量编码"选择目录，批量编码该目录下所有 `.txt` 文件。
   - 通过"文件 → 批量解码"对编码文件进行批量解码。

7. **配置选项**：
   - 点击"编辑 → 配置"（Ctrl+,）打开配置窗口。
   - **基本设置**：输出路径（数据目录/桌面）、默认文件名、自动打开文件夹等。
   - **编码设置**：默认编码方式、文件读写编码。
   - **高级选项**：是否自动清空输入、排序、覆盖确认、成功提示等。
   - 配置会自动保存。

### 命令行用法

```bash
# 加载指定文件并打开 GUI
encode.exe 名单.txt

# 显示版本信息
encode.exe -v

# 批量编码整个目录（Base64）
encode.exe --batch ./input_dir --output ./output_dir

# 批量编码整个目录（Hex）
encode.exe --batch ./input_dir --output ./output_dir --method hex
```

### 注意事项

1. 输入的名单应每行一个名字。
2. 支持多种文件编码（UTF-8 / GBK / GB2312 / GB18030 / UTF-16），自动尝试解码。
3. 默认情况下，编码后的文件保存在 `data/EncodeTool/encoded_files` 目录，也可以设置为保存在桌面。
4. 编码方式可在配置中切换，默认为 Base64。
5. 命令行批量模式适合大批量名单处理场景。

## 卸载工具

项目附带独立的卸载工具 `remove.py`（编译后为 `remove.exe`），提供两种卸载模式：

- **保留数据卸载**：仅移除程序文件，保留配置、日志和结果数据。
- **完全卸载**：移除程序文件及所有用户数据。

支持 Windows / Linux / macOS 三大平台。

## 安装和运行

>[!NOTE]
> 该项目主要支持 Windows 操作系统环境。对于 Linux 和 macOS 用户，可自行修改源码后手动编译。

1. 确保安装了 **Python 3.6 及以上** 版本环境（建议使用 3.10 及以上版本，推荐 3.12）。
2. 下载或克隆项目到本地。
3. 打开命令行或终端，进入项目目录。
4. 运行 `pip install pyinstaller` 安装 PyInstaller。
5. 运行根目录下的构建脚本，生成可执行文件：
   - `build_standard.bat` — 标准目录构建（生成多个 exe 及依赖文件到 `dist/` 目录）。
   - `build_onefile.bat` — 单文件构建（每个程序打包为单个独立 exe）。
   - `build_universe.bat` — 全量构建（合并构建标准版和单文件版，并复制所需资源文件）。
6. （可选）使用 **7z SFX Builder** 搭配 `SFX_config.txt`（需自行修改部分目录）打包自解压安装程序。

>[!NOTE]
> 7z SFX Builder 可从 [蓝奏云网盘](https://lzofevan.lanzn.com/ipqhQ3p9nk2d) 下载（密码:8kmj）。

>[!TIP]
> 或者可以直接下载 Release 版本并直接运行。
